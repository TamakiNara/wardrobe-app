# 開発ログ

このディレクトリは、個人開発の作業ログを月ごとに管理するための場所です。

## 目的

開発ログでは、以下を記録する。

- 何を作業したか
- 何を決めたか
- なぜそう判断したか
- どこで迷ったか
- 何が未解決か
- 次に何をすればよいか

単なる日記ではなく、作業を中断しても再開しやすくするための記録として扱う。

## ファイル構成

```txt
docs/dev-log/
  README.md
  _template.md
  YYYY-MM.md
```

例:

```txt
docs/dev-log/
  README.md
  _template.md
  2026-07.md
  2026-08.md
```

## 運用ルール

- 月ごとに1ファイル作成する
- ファイル名は `YYYY-MM.md` とする
- 新しい月のログは `_template.md` をコピーして作成する
- タスク専用ファイルは作成しない
- 現在有効なタスクは、月次ログファイル上部の「未来の自分へのタスク」に集約する
- 日別ログは、新しい日付を上に追加する
- 疲れている日はすべての項目を埋めなくてもよい
- 開発ログだけを仕様の正本にしない
- 確定した仕様・設計・実装方針は、必要に応じて既存の docs に反映する

## 月次ログの役割

月次ログには、次の2種類の情報を含める。

1. 月全体のタスク・未解決事項
2. 日ごとの作業記録

月次ログ上部には、現在有効なタスクを書く。
日別ログには、その日の作業内容・判断・迷い・次にやることを書く。

## 既存 docs との関係

開発ログは、作業経緯や判断過程を残す場所とする。
確定した仕様・DB設計・API設計・画面仕様・実装方針などは、必要に応じて既存の docs に反映する。

開発ログにだけ確定仕様を残すと、後から参照しづらくなるため、仕様の正本にはしない。

例:

```md
### 決めたこと

- wear log は `wear_logs` + `wear_log_items` で管理する
- 詳細は既存の仕様 docs に反映済み
```

未反映の場合は、タスクとして残す。

```md
- [ ] wear log の確定仕様を既存の仕様 docs に反映する
```
