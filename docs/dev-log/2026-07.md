### 決めたこと

- 開発ログ
  - 開発ログは `docs/dev-log/YYYY-MM.md` に作成する
  - テンプレートは `docs/dev-log/_template.md` に置く
  - 運用ルールは `docs/dev-log/README.md` に置く
  - 開発ログは作業経緯・判断理由・次タスクの記録場所とする
  - 確定仕様の正本は、開発ログではなく既存 docs 側に置く
  - 設計判断だけを切り出す新規ファイルは、現時点では作成しない
- docs 整理
  - `docs/README.md` は、初めて docs を読む人向けの入口にする
  - `docs/docs-map.md` は、仕様正本ではなく docs 構造の地図として扱う
  - docs の表記・命名ルールの正本は `docs/project/docs-writing-guidelines.md` に置く
- docs 更新用 skill
  - skill は詳細ルールを重複させず、正本 docs への誘導に絞る
