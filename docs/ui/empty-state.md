# Empty State Guidelines

空状態画面で「何もない」ときの見せ方を整理する資料。
この資料では、現時点で構想を詰めたい画面と、後で追加する空状態の候補をまとめる。

---

## 基本方針

- 空状態は「なぜないのか」と「次に何をすればよいか」が分かる構成にする
- 初回導線が必要な空状態では、CTA を明確に置く
- 「まだデータがない」場合と「条件に一致するデータがない」場合は分けて扱う
- エラー由来の非表示は、将来のエラー系 empty state として別論点で整理する

---

## 現時点で整理したい画面

### アイテム一覧

- アイテムが 1 件もないときの表示
- 初回登録への導線をどこまで強く出すか

### コーディネート一覧

- コーディネートが 1 件もないときの表示
- 作成前提となる item 登録への案内が必要かの整理

### 設定

- 設定画面で「空」と呼ぶべき状態があるかの整理
- 初回設定完了前の案内をどこまで空状態として扱うか

### ホーム

- ホームで表示するサマリーが空のときの見せ方
- 何を最初の行動として提案するかの整理

---

## 初期案

### アイテム一覧

- 未登録: 「まだアイテムが登録されていません。まずは 1 件追加してみましょう。」
- CTA: 「アイテムを追加する」
- 検索 / フィルタ結果 0 件: 「条件に一致するアイテムがありません。条件を変えてお試しください。」
- CTA: 「条件をクリアする」

### コーディネート一覧

- 未登録: 「まだコーディネートが登録されていません。手持ちのアイテムを組み合わせて作ってみましょう。」
- CTA: 「コーディネートを作成する」
- item が 0 件の場合: 「先にアイテムを登録して、組み合わせを作れる状態にしましょう。」
- CTA: 「アイテムを追加する」

### 設定

- 基本的には「空状態」としては扱わず、設定項目が読み込めない場合はエラー扱いとする
- 初回設定完了前の案内は、empty state ではなく onboarding として整理する

### ホーム

- items / outfits がどちらも 0 件のときは、「まずはアイテムを登録しましょう」を主案内にする
- items あり / outfits 0 件のときは、「次はコーディネートを作成してみましょう」と段階的に案内する
