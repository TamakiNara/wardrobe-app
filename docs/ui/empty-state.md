# Empty State Guidelines

空状態画面で「何もない」ときの見せ方を整理する資料。
この資料では、現時点で構想を詰めたい画面と、後で追加する空状態の候補をまとめる。

---

## 基本方針

- 空状態は「なぜないのか」と「次に何をすればよいか」が分かる構成にする
- 初回導線が必要な空状態では、CTA を明確に置く
- 「まだデータがない」場合と「条件に一致するデータがない」場合は分けて扱う
- エラー由来の非表示は、将来のエラー系 empty state として別論点で整理する

---

## 現時点で整理したい画面

### アイテム一覧

- アイテムが 1 件もないときの表示
- 初回登録への導線をどこまで強く出すか

### コーディネート一覧

- コーディネートが 1 件もないときの表示
- 作成前提となる item 登録への案内が必要かの整理

### 設定

- 設定画面で「空」と呼ぶべき状態があるかの整理
- 初回設定完了前の案内をどこまで空状態として扱うか

### ホーム

- ホームで表示するサマリーが空のときの見せ方
- 何を最初の行動として提案するかの整理
