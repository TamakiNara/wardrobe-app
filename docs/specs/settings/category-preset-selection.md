# Category Preset Selection Specification

## 概要

ユーザー登録直後に、カテゴリ表示設定の初期値を決めるための
**カテゴリプリセット選択画面** を表示する。

この画面は、`docs/specs/settings/category-settings.md` で定義している
カテゴリ表示設定の初期投入フローを補うための仕様である。

目的:

- 初回利用時に不要カテゴリを早めに減らす
- item / outfit 登録画面のカテゴリ候補を最初から使いやすくする
- 設定画面へ行かなくても初期設定を完了できるようにする

---

## 対象タイミング

- 新規ユーザー登録完了直後
- 初回ログイン状態が成立したあと
- ホームへ遷移する前

補足:

- 既存ユーザーには自動表示しない
- 将来的に「プリセット再適用」を実装する場合は別導線とする

---

## プリセット候補

選択肢は次の 3 つとする。

| 内部値 | UI 表示 |
| ------ | -------- |
| `male` | `メンズ` |
| `female` | `レディース` |
| `custom` | `カスタム` |

### `male` / `メンズ`

初期 OFF:

- スカート
- ワンピース
- オールインワン / サロペット
- パンプス

その他カテゴリは ON とする。

### `female` / `レディース`

すべて ON とする。

### `custom` / `カスタム`

すべて ON の状態から始める。

補足文言:

```text
カテゴリはあとから設定画面で変更できます
```

---

## 画面要件

画面には次を表示する。

- タイトル
- 補足説明
- 3 つのプリセット選択カードまたはラジオ UI
- 各プリセットの簡単な説明
- `次へ` ボタン
- `あとで設定` は設けない

現在の実装文言:

- タイトル: `カテゴリプリセットを選択`
- 説明: `使用するカテゴリの初期設定を選びます。` / `カテゴリはあとから設定画面で変更できます。`
- カード: `メンズ / レディース / カスタム`
- ボタン: `次へ`

判断メモ:

- 初期カテゴリ設定は item / outfit 体験に直結するため、登録直後に 1 回選んでもらう前提とする
- スキップを許すと、実質的に全カテゴリ ON のまま使い始める導線と二重化するため、初期仕様では非採用とする

---

## 選択後の遷移

### `male` / `female` を選んだ場合

- 選択したプリセットに応じて `visible_category_ids` の初期値を保存する
- 保存完了後、そのままホームへ遷移する

### `custom` を選んだ場合

- すぐには保存しない
- カテゴリ調整画面へ進ませる
- 初期表示は全カテゴリ ON とする
- ユーザーが保存を押した時点で `visible_category_ids` を保存する

---

## 未決定事項と今回の方針

既存仕様では次が未明確だったため、今回の docs 追加では以下で仮決めする。

### 1. 登録直後にホームへ行くか、プリセット選択へ行くか

方針:

- プリセット選択画面へ遷移する

### 2. スキップを許すか

方針:

- 初期仕様ではスキップ不可

### 3. `custom` 選択時に即保存するか

方針:

- 即保存しない
- 微調整画面で保存させる

---

## 実装メモ

- 登録 API 完了後の遷移先にこの画面を差し込む
- `male` / `female` は選択完了時に即保存
- `custom` は既存 settings UI の onboarding モードを使う

---

## 実装状況

- 登録完了後は `/register/category-preset` へ遷移する
- `male` / `female` は選択後にカテゴリ表示設定を保存してホームへ進む
- `custom` は settings の onboarding モードへ進み、保存後にホームへ進む
- 画面文言は `メンズ / レディース / カスタム` と `次へ` を使用する
