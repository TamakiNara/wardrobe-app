# インポート・エクスポート

## 概要

データをバックアップ・復元するための機能です。

Web UI:

- `/settings/import-export`
- 設定画面からエクスポート / インポートを実行できます

対象データ:

- アイテム（`items`）
- 購入検討（`purchase_candidates`）
- コーディネート（`outfits`）

データの対象範囲:

- ログインユーザーのデータのみを対象にします
- 他ユーザーのデータは含まれません
- import 実行時も、現在ログインしているユーザーの対象データのみ削除して復元します

`wear_logs` は今回の対象に含みません。

## エクスポート

エンドポイント:

- `GET /api/export`

説明:

- 対象データを JSON で取得します
- 画像は Base64 で JSON に含まれます

レスポンス例:

```json
{
  "version": 1,
  "exported_at": "2026-04-24T12:34:56+09:00",
  "owner": {
    "user_id": 1
  },
  "items": [],
  "purchase_candidates": [],
  "outfits": []
}
```

注意:

- 画像を Base64 で含むため、ファイルサイズが大きくなります

## インポート

エンドポイント:

- `POST /api/import`

説明:

- エクスポートした JSON を送信してデータを復元します

重要:

- import を実行すると、現在の対象データは全削除されます
- 復元後の ID は再採番されます
- `shape` / `spec` はエクスポート内容をそのまま使います
- 画像もあわせて復元されます
- バックアップファイルは作成したユーザー本人のみ復元できます
- 他ユーザーのバックアップファイルは復元できません
- 復元に失敗した場合、既存データは削除されません
- `owner.user_id` を含まない古い形式のバックアップファイルは復元できません

## 注意事項

- import すると現在のデータはすべて消えます
- `version` が異なるバックアップは失敗する可能性があります
- 古いバックアップは互換性がない可能性があります
- 画像が多い場合は処理時間が長くなります

## 推奨フロー

1. `GET /api/export` でバックアップを取得する
2. JSON を安全な場所へ保存する
3. `POST /api/import` で復元する

## 制限事項

- `wear_logs` は対象外です
- 差分更新はできません
- ZIP 形式ではなく JSON のみです
