## 運用案

- 実装済み spec には、原則として大きな backlog を本文へ積み上げず、このファイルへリンクする
- 各 spec には
  - current
  - planned
  - 要再判断
    までを最小限に残し、詳細な棚卸しは本ファイルへ寄せる
- 新しい後続タスクを追加するときは
  - 既存 backlog ID と重複しないか確認する
  - `出典 docs / 関連機能 / 優先度 / 依存関係 / すぐ実装できるか`
    を埋める
