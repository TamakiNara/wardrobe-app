# Search, Filter, and Sort Specification

一覧画面で使う検索・絞り込み・並び順の共通仕様を整理する資料。
この資料では、対象画面、検索対象、絞り込みの考え方、並び順、0件時の空状態の扱いを定義する。

---

## 対象画面

- アイテム一覧
- コーディネート一覧
- 着用履歴一覧

---

## 基本方針

- 検索と絞り込みは分けて扱う
- 検索はテキスト入力とする
- 絞り込みは選択式条件とする
- 並び順は単一選択とする
- フィルタ結果が 0 件になってもよい
- 「データ自体が 0 件」と「絞り込み結果が 0 件」は別の空状態として扱う

---

## 検索

### 定義

- 検索はキーワードのテキスト入力で行う
- 部分一致の単純検索とする
- 初期仕様では高度な検索構文は持たない

### 検索対象項目

- アイテム名
- ブランド名
- メモ

### 画面ごとの適用方針

#### アイテム一覧

- アイテム名
- ブランド名
- メモ

#### コーディネート一覧

- コーディネートに含まれるアイテム名
- コーディネートに含まれるブランド名
- コーディネートのメモ

補足:

- コーディネート自体に名前が入る仕様を将来追加した場合は、その名称も検索対象に含める

#### 着用履歴一覧

- 着用記録のメモ
- 紐づく item 名
- 紐づく brand 名

補足:

- 記録時点のスナップショット情報を基準に検索できる形が望ましい

---

## 絞り込み

### 定義

- 絞り込みは選択式条件で行う
- 選択 UI はチェックボックス、ラジオ、セレクトなどを想定する
- 検索とは別 UI として扱う

### 初期方針

- 画面ごとに必要な絞り込み条件は今後個別に定義する
- 本資料ではまず「検索と絞り込みを分離する」ことを正本とする
- フィルタ適用結果が 0 件でも許可する

### 空状態

#### データ自体が 0 件

- まだ 1 件も登録されていない状態
- 初回登録導線を強く出す

#### 絞り込み結果が 0 件

- データは存在するが、現在の絞り込み条件に一致しない状態
- 条件クリアや条件見直しの導線を出す

---

## 並び順

### 定義

- 並び順は単一選択とする
- 初期候補は次の 3 種を共通候補とする
  - 新しい順
  - 古い順
  - 名前順

### 初期値

- アイテム一覧: 新しい順
- コーディネート一覧: 新しい順
- 着用履歴一覧: 日付の新しい順

### 補足

- 着用履歴一覧では「新しい順」は `wear_date` の降順を意味する
- 同日の複数件をどう並べるかは、将来 `created_at` などの副キーで補完する

---

## 画面別まとめ

### アイテム一覧

- 検索: アイテム名 / ブランド名 / メモ
- 絞り込み: 選択式条件を今後定義
- 並び順初期値: 新しい順

### コーディネート一覧

- 検索: 含まれるアイテム名 / ブランド名 / メモ
- 絞り込み: 選択式条件を今後定義
- 並び順初期値: 新しい順

### 着用履歴一覧

- 検索: 記録メモ / 紐づく item 名 / brand 名
- 絞り込み: 選択式条件を今後定義
- 並び順初期値: 日付の新しい順
- 0件時は「記録自体がない」と「条件で絞り込みすぎた」を分けて扱う

---

## 今後の検討ポイント

- 各画面で必要な絞り込み条件の具体化
- URL クエリと状態管理の持ち方
- 検索・絞り込み・並び順をどこまで組み合わせ可能にするか
- API 側で受ける query parameter の設計
- 空状態 UI の文言と導線
